Core design philosophy
The system does the knowing — the customer makes decisions. Every touchpoint is designed to surface a pre-computed answer for confirmation, not a blank form for data entry. AI automation handles classification, field inference, discrepancy detection, and anomaly monitoring. Humans are engaged only for judgment calls: tier overrides, low-confidence AI suggestions, batch YTD sign-offs, and the decommission gate.

Employee tier routing — completeness-based classification

Tier 1
Active, full data available
All 9 required fields present: SSN, address, W-4 filing status, YTD wages, YTD taxes, deduction items, pay schedule, last pay date, employment status.
Migrate as paycheck object — fully linked to QBO employee record
Tier 2
Active, partial data
Employee is active but one or more required fields are missing or unconfirmed — stale W-4, missing local tax ID, unmapped deduction items, or non-standard pay types.
Guided remediation → migrate as paycheck after gap-fill confirmed
Tier 3
Inactive or prior-period only
Employee no longer active, or only prior-year paychecks exist with insufficient data to meet QBO Payroll's minimum requirements for a live paycheck record.
Preserved paycheck archive — structured, queryable, re-linkable

Pre-Migration Phase
Establish a complete picture of the data landscape before a single record moves. The goal is to eliminate every surprise — file health, employee completeness, payroll item compatibility, and reconciliation anchors are all resolved here. The customer leaves this phase with a signed-off readiness report, not an open question.

AI automation scope in pre-migration
AI reads and scores QBDT files autonomously. It infers missing W-4 details from 6-month withholding patterns, semantically maps payroll item names to QBO equivalents, and auto-exports baseline reports with zero customer input. The customer's cognitive task is reduced to reviewing and confirming a pre-computed state — not assembling it.

Employee completeness scoring
Every employee record is automatically scored across 9 required fields and assigned to Tier 1, 2, or 3 — with manual override available for edge cases.
AI automation
Completeness algorithm checks 9 fields per employee: SSN, address, W-4 filing status, YTD wages, YTD taxes, deduction items, pay schedule, last pay date, employment status
Tier assignment is automatic — Tier 1 (all fields present), Tier 2 (active, gaps found), Tier 3 (inactive or insufficient for QBO Payroll requirements)

Table view sorted by tier — Tier 1 (green), Tier 2 (amber), Tier 3 (coral). Missing fields shown as inline badges per row, not buried in detail screens
Filter/sort by tier, department, last pay date. Bulk-select for shared issue resolution across multiple employees

AI inference Confirm / override
AI gap-fill suggestions

AI automation
W-4 filing status inferred from 6-month withholding patterns. Address inferred from check delivery history. Deduction items matched from paycheck line items
Confidence score shown per suggestion — high confidence (≥85%) auto-approved in bulk, low confidence flagged amber for explicit human review

During Migration Phase
Progressive commitment through tiered processing — each batch confirmed before the next begins. No all-or-nothing migration window. The check-to-paycheck intercept layer ensures no paycheck is silently converted to a check. Every exception is grouped, explained in plain English, and resolved with a single suggested action.

AI intercept Data recovery
Check-to-paycheck intercept
If the native QBO tool converts any paycheck to a check, the intercept layer catches it, re-matches it to QBDT payroll data, and re-posts it as a full paycheck object.
AI automation
Identifies check-converted paychecks by matching check date + net amount + employee ID + period end date against the QBDT payroll export — multi-field matching prevents false positives
Matched records re-enriched with payroll item breakdowns from preserved QBDT data and re-posted as paycheck objects in QBO Payroll — net amount stays identical, payroll structure is restored
UX design
Customer sees "X paychecks recovered from check conversion" — a success signal, not a warning. This communicates that the system is actively protecting their payroll data integrity
Unmatched check conversions (no QBDT source found) are surfaced in the exception queue with the check details and a guided path to manual reconciliation

Migration progress dashboard
Live counters for each tier — no black-box waiting. Time estimates update in real time. Migration can be paused between payroll cycles and resumed without data loss.
UX design
Live counters: Tier 1 migrated / Tier 2 in remediation / Tier 3 archived / exceptions flagged. Customer knows exactly where they are at every moment
Time estimate updates in real time based on actual processing speed. If migration runs longer than expected, customer is notified proactively — not surprised

AI clustering Plain language
Exception surfacing
Errors grouped by root cause, explained in plain English, each with a single suggested action. Never raw error codes. Blocking vs advisory vs informational classification.
AI automation
Error clustering: if 8 employees have the same root cause, they're shown as one group with a single-action fix — not 8 separate alerts. Reduces exception-handling cognitive load by up to 80% on large files
Error categorization: Blocking (must fix before migration completes), Advisory (can proceed but should review), Informational (logged, no action needed)
UX design
Every error shows: what happened / why it happened / one suggested action. Example: "Sarah Chen: missing state tax ID — click here to add it." No error codes, no technical jargon
Advisory errors include an "explain this difference" link that shows AI-generated context — e.g. known QBO rounding behavior, FIFO vs average cost inventory change

Human sign-off AI explanation
YTD validation checkpoint
After each employee batch, a side-by-side QBDT vs QBO YTD comparison is presented for customer sign-off — with AI explaining any delta in plain English.
Data integrity
Side-by-side comparison per batch: QBDT YTD vs QBO YTD for gross wages, federal tax, state tax, FICA, each deduction line. Line-item diff, not just totals
Matches shown green (no action), discrepancies shown amber with delta amounts. Customer signs off per batch — incremental review, not one overwhelming final screen
AI automation
AI explains each delta in plain English: "The $12.50 difference in John's federal tax is due to QBO's daily rounding method vs QBDT's per-period method — within IRS tolerance"
Expected differences (rounding, FIFO/avg cost, audit trail exclusion) are pre-classified and explained automatically — customer only investigates genuinely unexpected items
◉
Post-Migration Phase
Confidence through evidence, not faith. The parallel-run monitor replaces the subjective "I feel ready" with a measurable signal — 5 consecutive clean days before QBDT decommission is recommended. Anomaly detection on the first 3 live QBO payroll runs catches problems before employees experience them. The sign-off report closes the loop with a dated, auditable record of what moved, what was preserved, and what was resolved.

Pattern detection Pre-submit alert
Anomaly detection alerts
AI monitors the first 3 live QBO payroll runs against QBDT historical patterns — statistical outliers surface before payroll is submitted, not after employees report issues.
AI automation
After each of the first 3 live payroll runs, AI compares gross wages, tax withholdings, net pay distributions, and deduction totals against the last 6 months of QBDT payroll patterns
If a run is statistically anomalous (e.g. total net pay is 15% lower than rolling average, or a specific deduction is missing across all employees), an alert fires before payroll is submitted
Human touchpoint
Customer can mark an anomaly as expected (e.g. seasonal workforce reduction, one-time bonus period) to train the monitor — reduces false positives without requiring IT or ProAdvisor involvement
Anomaly history is retained in the migration audit trail — if a payroll error surfaces later (employee complaint, tax notice), the detection record provides a forensic timeline

Cognitive load reduction map

Across all 18 touchpoints, only 4 require a mandatory human decision. The table below maps each touchpoint to its interaction mode and the cognitive task it replaces — showing where AI reduces memory-and-retrieval tasks to confirmation tasks.

Core design principles — reducing cognitive burden
1
Never ask a customer to retrieve information — only ask them to confirm or override what the system already found
2
Progressive commitment — migration is a sequence of small confirmed steps, not one all-or-nothing window
3
Group exceptions by root cause — one decision for 15 employees beats 15 identical decisions
4
Confidence through evidence — decommission is gated on a measurable signal, not a feeling
5
Every error gets: what happened / why it happened / one suggested action — no error codes, no jargon
6
AI fills blanks, humans make judgment calls — the boundary is always explicit and never crossed silently
